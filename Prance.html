為什麼選擇 FPGA 在交易系統中？

極致延遲表現

一般 CPU 必須逐筆指令執行，FPGA 則以硬體邏輯平行處理行情、風控與下單流程， 可將延遲從微秒級進一步壓低到奈秒等級，特別適合高頻與 DMA 交易。

Tick-to-Trade 全程硬體化

行情解碼、Order Book 建構、策略判斷、風控與封包送出都在 FPGA Pipeline 中完成， 只將非即時性處理保留在軟體端，達到效能與彈性的最佳平衡。

維持熟悉的開發模式

使用 C/C++ 撰寫策略，透過我們的工具鏈轉成 FPGA 邏輯，不需直接撰寫 RTL， 也不必接觸低階硬體細節，讓既有量化團隊可以快速上手。

多市場、多商品支援

彈性的 Feed Handler 與 Order Entry 模組，可快速對接不同交易所協定（含 FIX）， 支援股票、期貨、選擇權等商品，配合客戶需求擴充。

系統架構與資料流

整體架構分為「市場與來源」、「FPGA Tick-to-Trade 核心」以及「軟體與行動平台」三個層次， 自多來源行情與委託開始，經由 FPGA 完成解碼、策略與風控，再送往交易所並回饋至帳務與前端介面。

三層架構說明

  1. 市場與客戶端
    來自各交易所與資訊供應商的多路行情，以及 DMA 大戶、券商自營部、 行動 / Web App 的委託與參數輸入。
  2. FPGA Tick-to-Trade 核心
    高速網路介面與 TCP Offload、Feed Handler 解碼、 Order Book 引擎、策略引擎與即時風控， 最終產生符合各交易所規範的委託電文。
  3. 軟體後台與行動平台
    交易 API 負責設定策略參數與監控 FPGA 狀態，帳務中台提供即時損益與報表， Web / App 讓大戶與一般投資人都能享有低延遲體驗。

導入與開發流程：三個步驟，把策略搬進 FPGA

Step 1

撰寫並驗證策略

使用 C/C++ 在我們提供的策略範本上撰寫交易邏輯， 輸入包含 Order Book 狀態、解碼後行情與自訂參數。

先在模擬或回測環境驗證績效與風控邏輯，確保策略穩定可靠。

Step 2

一鍵編譯到 FPGA

透過我們的工具鏈在您的伺服器上完成 HLS 編譯與 Bitstream 產生， 策略程式碼不需離開內部環境，滿足資訊安全要求。

支援反覆調整與重新部署，讓團隊可以持續優化策略。

Step 3

整合既有系統並啟動交易

使用 C++ / REST API 將 FPGA 核心與既有風控、前台或 OMS 整合， 後台可監控 FPGA 狀態、策略統計與風險參數。

Web / App 介面可供大戶與一般投資人使用，享受同樣的低延遲體驗。
